# PyCat v0.0.4

> Release notes for GitHub Release

## Suggested release title

`PyCat v0.0.4`

## Suggested tag

`v0.0.4`

## Release body

PyCat v0.0.4 在 v0.0.3 的 **Capability + Subagent + 三层 Memory** 基础上，集中完成了四大核心升级：**Artifact 内容管理体系**、**子 Agent 追踪可视化**、**搜索多提供商架构**与**工具权限治理集中化**，同时对工具选择策略、嵌套调用追踪、终端配置和提示词上下文组装进行了深度重构。

### 🧭 项目定位

在 Claude Code（终端编码 Agent，Rust/Shell/TS）、Codex（终端编码 Agent，Rust 96%）、Cherry Studio（Electron 桌面客户端，TS + Electron ~200MB）、Chatbox（Electron 桌面客户端，TS + Electron ~200MB）、OpenClaw（终端编码 Agent）等成熟项目之间，**PyCat 选择了一条不同的技术路线**：

| 维度 | 主流方案 | PyCat 方案 |
|------|---------|------------|
| **语言** | TypeScript + Rust + Electron + Bun（3-5 种） | **Python 单语言** |
| **桌面 UI** | Electron（捆绑 Chromium + Node.js，~200MB+） | **PyQt6 原生控件（Nuitka 编译，~80MB）** |
| **编译产物** | ASAR 打包 + 解释执行 | **Nuitka 原生机器码编译** |
| **IM 频道** | 无 | **QQ / 微信 / 飞书 / Telegram 四平台** |
| **扩展方式** | IPC / SDK / 插件子系统 | **直接 Python import** |
| **能力抽象** | 无统一能力层 | **Capability + Subagent 双层委托** |
| **Memory** | 会话级或云端 | **Session / Workspace / Global 三层本地记忆** |
| **工具治理** | 简单开关 | **类别默认 + 单工具覆盖的两层权限模型** |
| **Artifact 管理** | 无 | **Session 级 artifact 落盘 + 生命周期管理** |

**PyCat 的技术特点：**

1. **Python 单一技术栈** — 不需要在多种语言间切换，降低上手和维护门槛。
2. **Nuitka 原生编译，包体积更小** — 不捆绑浏览器内核，~80MB 即可独立运行。
3. **四大 IM 频道内置** — QQ、微信、飞书、Telegram 均已接入基础功能。
4. **原生 Python 可扩展** — Skills、Tools、MCP、Channels 全部在 Python 中直接实现。
5. **Capability + Subagent 双层委托** — 固定用途能力自动暴露为工具，复杂任务通过默认子 Agent 委托，路由清晰可控。
6. **三层 Memory 架构** — Session（当前会话）、Workspace（当前项目）、Global（跨项目用户画像）分层记忆，按需注入上下文。
7. **Artifact 内容管理** — 会话级输出物支持 frontmatter 元数据、生命周期状态与自动落盘，替代零散文档管理。
8. **架构轻量，迭代迅速** — 从 v0.0.1 至今已完成四大频道落地、SDK 轻量化、能力体系、子 Agent 委托、记忆分层、Artifact 管理与工具治理集中化。

> ⚠️ **诚实地说明**：PyCat 目前处于早期阶段，功能深度、稳定性、社区规模远不及上述成熟项目。Cherry Studio 在可扩展性和响应格式覆盖上更完善，Chatbox 在移动端体验上更成熟。如果你的目标是找一个生产就绪的桌面 AI 工具，建议优先考虑它们。如果你对 Python + Nuitka + IM 频道 + 分层能力委托这条技术路线感兴趣，欢迎来一起完善。

## ✨ Highlights

- **Artifact 内容管理体系**：将旧的 `document` 概念全面重构为 `artifact`，引入 `manage_artifact` 工具替代旧文档管理。支持 frontmatter 元数据、生命周期状态（draft / approved / final / archived）、引用关系、摘要索引，自动落盘到 `.pycat/sessions/<id>/artifacts/`，实现会话输出物的结构化持久化。
- **子 Agent 追踪面板**：新增 `SubagentTraceWidget` 组件，支持展开/折叠子 Agent 执行轨迹，实时展示状态、消息数、耗时及摘要。嵌套工具调用链路可视化，父 Agent 可直观观察 `subagent__read_analyze`、`subagent__search`、`subagent__custom` 的内部执行过程。
- **搜索多提供商架构**：重构搜索服务，支持 **DuckDuckGo**（零配置）、**Tavily**（AI 优化结果）、**Brave**（隐私优先，免费额度）、**SearxNG**（自托管）四种搜索提供商。统一 `BaseSearchProvider` 抽象与 `SearchProviderFactory`，遵循 Cherry Studio 的 provider pattern，配置管理更清晰。
- **工具权限与选择治理集中化**：移除旧的分散布尔开关，引入 `ToolSelectionPolicy` 与 `ToolCatalog`，支持请求时按类别/工具/来源三层过滤；重构 `ToolPermissionPolicy` + `ToolPermissionResolver`，将权限判断集中到一个统一入口，支持运行时按类别默认 + 单工具覆盖的两层解析策略。
- **嵌套工具调用追踪**：增强子任务处理与工具调用结果管理，支持嵌套工具调用追踪。子 Agent 内部调用的工具结果可被正确捕获、展示和追溯，解决了子 Agent 工具执行"黑盒化"问题。
- **终端设置页面**：新增**终端**设置页，支持 bang 命令处理方式配置（shell / agent 模式）、Shell 后端选择（CMD / PowerShell / WSL）、输出编码（UTF-8 / GB18030 / 自动检测）、可执行文件路径及环境变量继承设置。
- **提示词上下文提供者体系化**：将提示词组装拆分为 `core/prompts/providers/` 子模块（artifact、environment、memory、summary、todo），支持模块化上下文注入与按需裁剪，系统提示词构建从 monolithic 走向组合式。
- **待办事项工具**：新增 `manage_todo` 工具，支持会话级待办事项的创建、更新、完成、删除与列出。待办作为轻量级进度状态，与 `manage_artifact`（输出物）和 `manage_memory`（持久事实）形成清晰分工。
- **消息树视图模型**：新增 `ui/view_models/message_tree.py`，为对话分支管理提供数据模型支持，是后续多分支会话 UI 的数据基础。

## 🔧 从 v0.0.3 以来的主要变更

### 新增

- **Artifact 内容管理**：`core/state/services/artifact_service.py` + `core/tools/system/artifact_tools.py`，`manage_artifact` 支持 create / read / update / delete / list / append 六种操作
- **子 Agent 追踪面板**：`ui/widgets/` 新增 `SubagentTraceWidget`，支持深/浅主题样式与左侧边框高亮
- **搜索多提供商**：`services/search_providers/` 包，包含 `duckduckgo.py`、`tavily.py`、`brave.py`、`searxng.py`、`factory.py`、`base.py`
- **终端设置页面**：`ui/settings/pages/terminal_page.py`，支持 bang 命令行为、Shell 后端、编码、WSL 发行版配置
- **工具目录与选择策略**：`core/tools/catalog.py` 引入 `ToolSelectionPolicy`、`ToolDescriptor`、`ToolAvailabilityContext`
- **提示词上下文提供者**：`core/prompts/providers/` 子模块，包含 `artifact.py`、`environment.py`、`memory.py`、`summary.py`、`todo.py`
- **消息树视图模型**：`ui/view_models/message_tree.py`，支持对话树形结构的数据模型
- **待办工具**：`core/tools/system/todo_tools.py` 引入 `manage_todo`，支持会话级轻量进度跟踪

### 优化

- **工具权限治理集中化**：`core/tools/permissions.py` 重构为 `ToolPermissionPolicy` + `ToolPermissionResolver`，统一权限判断入口
- **结果管道配置**：`core/tools/result_pipeline.py` 阈值与预览策略可配置化，支持按工具类型定制落盘策略
- **文件操作工具重命名**：`write_to_file` → `file_write`、`read_file` → `file_read`，命名更清晰一致
- **搜索服务重构**：`services/search_service.py` 统一适配多提供商架构，`models/search_config.py` 同步更新
- **系统提示词组装**：`core/prompts/system_builder.py` 从 monolithic 模板拆分为组合式 provider 调用
- **频道运行时**：回复与会话验证链路改进，工具调用实时可见性持续优化
- **默认记忆源扩展**：默认记忆源从 `("session", "workspace")` 扩展为 `("session", "workspace", "global")`
- **对话设置重构**：移除 `enable_mcp` / `enable_search` 等分散字段，统一由 `ToolSelectionPolicy` 管理
- **UI 布局与样式**：主窗口与设置页截图资源更新，输入区域和能力列表样式持续调优

### 修复

- 嵌套工具调用结果丢失问题 — 子 Agent 内部工具执行结果现在可被正确捕获和展示
- 旧 `document` 相关引用全面清理 — `core/tools/system/document_tools.py` 已移除，`document_service.py` 已删除
- 搜索配置与 MCP 代理兼容性修复
- `manage_memory` 工具引用明确化，避免与 `manage_state` 职责重叠
- 超长工具结果在嵌套调用场景下的重复落盘问题

## 📝 Notes

- 本版本已包含 `assets/`、`LICENSE`、`README.md`、`README_zh.md` 及运行时依赖。
- 当前提供的发布资产为 **Windows x64** 独立包。
- 由于可执行文件未进行代码签名，Windows 首次运行时若出现 SmartScreen 提示，属于常见现象，可在确认来源可信后继续运行。
- **Artifact 是新增的核心抽象**，建议参考 `docs/architect/` 系列文档了解 artifact 与 memory 的职责边界：artifact 用于会话输出物和工作材料，memory 用于持久可复用的事实和偏好。
- **搜索多提供商默认不启用**，需在设置的「搜索」页中配置至少一个提供商并填入对应 API Key（DuckDuckGo 除外，零配置即可使用）。
- **工具命名已进一步规范化**，文件操作工具从 `write_to_file` 统一为 `file_write`，旧外部脚本或 MCP 配置若引用旧工具名需同步更新。
- 当前仍在快速迭代中，虽然还有很多不足，但进展迅速。欢迎所有人加入一起优化项目。
