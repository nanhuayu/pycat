# PyCat v0.0.3

> Release notes for GitHub Release

## Suggested release title

`PyCat v0.0.3`

## Suggested tag

`v0.0.3`

## Release body

PyCat v0.0.3 在 v0.0.2 的**四平台 IM 频道 + Chat / Agent / Tools 桌面一体化**基础上，集中完成了三大核心升级：**Capability（能力）系统**、**Subagent（子 Agent）委托体系**与**三层 Memory（记忆）架构**，同时对工具命名规范、模型能力画像、工具结果处理管线、权限模型和架构文档进行了系统性重构。

### 🧭 项目定位

在 Claude Code（终端编码 Agent，Rust/Shell/TS）、Codex（终端编码 Agent，Rust 96%）、Cherry Studio（Electron 桌面客户端，TS + Electron ~200MB）、Chatbox（Electron 桌面客户端，TS + Electron ~200MB）、OpenClaw（终端编码 Agent）等成熟项目之间，**PyCat 选择了一条不同的技术路线**：

| 维度 | 主流方案 | PyCat 方案 |
|------|---------|------------|
| **语言** | TypeScript + Rust + Electron + Bun（3-5 种） | **Python 单语言** |
| **桌面 UI** | Electron（捆绑 Chromium + Node.js，~200MB+） | **PyQt6 原生控件（Nuitka 编译，~80MB）** |
| **编译产物** | ASAR 打包 + 解释执行 | **Nuitka 原生机器码编译** |
| **IM 频道** | 无 | **QQ / 微信 / 飞书 / Telegram 四平台** |
| **扩展方式** | IPC / SDK / 插件子系统 | **直接 Python import** |
| **能力抽象** | 无统一能力层 | **Capability + Subagent 双层委托** |
| **Memory** | 会话级或云端 | **Session / Workspace / Global 三层本地记忆** |
| **工具治理** | 简单开关 | **类别默认 + 单工具覆盖的两层权限模型** |

**PyCat 的技术特点：**

1. **Python 单一技术栈** — 不需要在多种语言间切换，降低上手和维护门槛。
2. **Nuitka 原生编译，包体积更小** — 不捆绑浏览器内核，~80MB 即可独立运行。
3. **四大 IM 频道内置** — QQ、微信、飞书、Telegram 均已接入基础功能。
4. **原生 Python 可扩展** — Skills、Tools、MCP、Channels 全部在 Python 中直接实现。
5. **Capability + Subagent 双层委托** — 固定用途能力自动暴露为工具，复杂任务通过默认子 Agent 委托，路由清晰可控。
6. **三层 Memory 架构** — Session（当前会话）、Workspace（当前项目）、Global（跨项目用户画像）分层记忆，按需注入上下文。
7. **架构轻量，迭代迅速** — 从 v0.0.1 至今已完成四大频道落地、SDK 轻量化、能力体系、子 Agent 委托、记忆分层与架构文档体系化。

> ⚠️ **诚实地说明**：PyCat 目前处于早期阶段，功能深度、稳定性、社区规模远不及上述成熟项目。Cherry Studio 在可扩展性和响应格式覆盖上更完善，Chatbox 在移动端体验上更成熟。如果你的目标是找一个生产就绪的桌面 AI 工具，建议优先考虑它们。如果你对 Python + Nuitka + IM 频道 + 分层能力委托这条技术路线感兴趣，欢迎来一起完善。

## ✨ Highlights

- **Capability 能力系统**：引入 `CapabilityConfig` 配置化抽象，将提示词、模型、运行模式、工具组和执行预算统一封装为**可复用能力**。每个启用的能力自动注册为独立的 `capability__*` 工具（如 `capability__translate`、`capability__summarize_text`），通过 child-task scheduler 执行，不再是隐藏的 wrapper 或黑盒路由器。
- **Subagent 委托体系**：建立三个语义清晰的默认子 Agent — `subagent__read_analyze`（只读多文件分析）、`subagent__search`（搜索与事实核查）、`subagent__custom`（运行时自定义委托）。父 Agent 可显式收窄子 Agent 的工具权限和输出目标，委托边界明确。
- **三层 Memory 架构**：新增 **Global Memory** 支持，从 `~/.PyCat/memory/` 加载跨工作区持久记忆，入口文件为 `SOUL.md`，主题文件采用 `memory__<topic>.md` 命名规范。`manage_state` 新增 `memory_tier` 参数，支持 `session` / `workspace` / `global` 三级写入。对话设置页可独立控制每层记忆的注入开关。
- **工具命名体系重构**：统一采用语义化双下划线命名，提升模型识别度和用户可读性 — `skill__load`、`skill__read_resource`、`capability__summarize_text`、`subagent__read_analyze`。旧命名已迁移，不再保留兼容壳。
- **工具结果处理管线**：替换旧的 monolithic `_maybe_spill_tool_result`，引入 `ToolResultPipeline` 状态机。超长工具结果（>12,000 字符）自动保真落盘到 `.pycat/tool-results/`，在 prompt 中保留首尾预览、字符数、完整文件路径和 `read_file` 恢复提示，避免上下文膨胀。
- **模型能力画像**：新增 `ModelProfile` 数据结构，支持为每个模型配置上下文窗口、工具调用、视觉、推理风格（`reasoning_style`）、推理力度（`reasoning_effort`）、思考预算（`thinking_budget_tokens`）等能力画像，实现多模型角色编排与精准请求构建。
- **对话设置全面升级**：新增**记忆策略**分组（session / workspace / global 三选一或多选）和**频道策略**分组（allowlist / trust / notice 会话级 policy）。模式切换时自动联动调整 MCP、搜索、思考等功能开关的可用态和默认值。
- **权限模型增强**：新增 `delegate` 权限类别（专用于 `capability__*` 和 `subagent__*` 工具），形成 read / edit / command / delegate / misc 五类风险分级，每层支持按类别默认 + 单工具覆盖的两层解析策略。
- **架构文档体系化**：新增并重构 `docs/architect/` 完整文档系列（ability.md、agent.md、capabilities.md、command.md、design.md、memory.md、model.md、session.md、tools.md、ui.md），覆盖每个核心子系统的数据结构、运行时链路、扩展规范与维护规则。

## 🔧 从 v0.0.2 以来的主要变更

### 新增

- **Capability 能力系统**：`core/capabilities/` 整套子系统（`types.py`、`manager.py`、`defaults.py`、`exposure.py`），支持配置化能力定义与动态工具注册
- **内置能力集合**：`capability__prompt_optimize`（提示词优化）、`capability__title_extract`（标题提取）、`capability__translate`（翻译）、`capability__context_compress`（上下文压缩）、`capability__summarize_text`（长文总结）等
- **Capability 工具适配层**：`core/tools/system/capability_tools.py`，负责将能力配置渲染为独立工具并调度子任务
- **默认 Subagent 工具**：`subagent__read_analyze`、`subagent__search`、`subagent__custom`、`attempt_completion`、`switch_mode`，位于 `core/tools/system/multi_agent.py`
- **全局 Memory 支持**：`MemoryService.load_global_memory()` 从 `~/.PyCat/memory/` 读取跨工作区记忆，支持 `SOUL.md` 入口和链接解析
- **Memory 写入分级**：`manage_state` 支持 `memory_tier="session"|"workspace"|"global"`，分别写入会话状态、工作区 `.pycat/memory/` 或全局 `~/.PyCat/memory/`
- **工具结果处理管线**：`core/tools/result_pipeline.py` 引入 `ToolResultPipeline`、`ResultHandle`、`ResultDisposition`，实现超长结果自动落盘与可恢复预览
- **模型能力画像**：`models/model_profile.py` 引入 `ModelProfile`，支持上下文窗口、视觉、工具调用、推理风格等维度画像
- **对话设置记忆策略**：`ConversationSettingsDialog` 新增 session / workspace / global 记忆来源独立开关
- **对话设置频道策略**：支持 allowlist / trusted sources / notice policy 的会话级细粒度控制
- **Provider 配置对话框重构**：支持模型能力画像编辑与 `provider|model` 统一引用格式
- **架构文档系列**：`docs/architect/` 下新增/重构 10 篇子系统架构文档

### 优化

- **工具命名规范化**：`load_skill` → `skill__load`、`read_skill_resource` → `skill__read_resource`，系统工具全面采用语义化双下划线命名
- **权限模型细化**：`ToolPermissionConfig` 新增 `delegate` 类别，支持 `capability__*` / `subagent__*` 工具的独立权限控制
- **系统提示词增强**：`system_builder.py` 注入能力工具使用指南、状态工作流指引（按 mode 定制）和可用技能目录
- **子任务上下文裁剪**：子 Agent 默认只获得父 Agent 裁剪后的上下文，避免信息过载
- **对话设置联动**：切换 mode 时自动调整 MCP、搜索、思考开关的可用态与默认值
- **LLM 请求构建增强**：`request_builder.py` 对 reasoning 模型清洗缺失 reasoning 字段的脏历史，避免 `invalid_request_error`
- **Channel 运行时增强**：回复与会话验证改进，频道触发的工具调用实时可见性持续优化
- **UI 布局与样式**：输入区域和能力列表样式调整、按钮尺寸优化、思考内容显示改进、流式响应 UI 优化

### 修复

- 旧 `new_task` / `run_sub_agent` 入口已完全移除，统一为 `capability__*` / `subagent__*` 命名体系
- 旧 `capability__summarize_long_text` 等过时引用已清理，文档和提示词中不再引导使用旧入口
- `ChannelConversationBindingStore` 测试与真实数据目录隔离（延续）
- 超长工具结果不再直接塞入上下文导致 token 溢出，改为保真落盘 + 预览提示

## 📝 Notes

- 本版本已包含 `assets/`、`LICENSE`、`README.md`、`README_zh.md` 及运行时依赖。
- 当前提供的发布资产为 **Windows x64** 独立包。
- 由于可执行文件未进行代码签名，Windows 首次运行时若出现 SmartScreen 提示，属于常见现象，可在确认来源可信后继续运行。
- **Capability 与 Subagent 是新增的核心抽象**，建议参考 `docs/architect/capabilities.md` 了解能力工具与子 Agent 工具的路由边界。
- **Global Memory 默认不自动注入**，需在对话设置的「记忆策略」中手动启用 `全局记忆`。
- 工具命名已统一为双下划线风格，旧外部脚本或 MCP 配置若引用旧工具名需同步更新。
- 当前仍在快速迭代中，虽然还有很多不足，但进展迅速。欢迎所有人加入一起优化项目。
